# 任务卡 Slice s04 - 风险即时提示锁存与超时自动结束防护

## 1. 本卡要交付的用户能力
用户在 `RUNNING` 中出现碰撞/压线/超时/输入越界时，HUD 会即时提示并锁存风险，超时会自动结束本局，输入越界会被钳制且会话不中断。

## 2. 本卡为什么现在做
- 解决核心缺口：当前已能真实结算，但“运行中风险治理”链路不完整。
- 消灭最大风险：无锁存与超时兜底会导致“可结束但不可控”的体验和判定不稳定。
- 对应 first playable 阶段：在 s03 基础上补齐“稳定可练”的运行期防护。

## 3. 触达的闭环链路
- Trigger: `RUNNING` 期间发生碰撞、压线、超时、输入越界。
- Input: 风险事件、控制输入、会话时间、当前姿态与场景约束。
- Runtime/State: `RUNNING` 持续运行并记录风险；超时触发 `RUNNING -> SETTLING -> DONE`。
- Core Logic: `risk-recorder + session-end-policy + input-guard -> session-controller -> HUD/result-panel`。
- Derived Facts / Settlement: 风险锁存事实、超时失败事实、钳制后的有效控制输入。
- UI Output: HUD 风险区即时提示且锁存，结果面板显示自动结束结果。

## 4. 影响的状态机
- 当前涉及状态：`RUNNING`、`SETTLING`、`DONE`。
- 新增/修改的状态迁移：
- 风险事件不直接中断输入链路，但写入锁存风险状态。
- 达到超时阈值时自动触发 `RUNNING -> SETTLING -> DONE`。
- `DONE -> READY`（重试）时清空风险锁存。
- 不在本卡处理的状态：四场景循环切换与全量场景差异验收（s05）。

## 5. 影响的数据流
- 输入数据：控制输入（方向/油门）、风险事件、会话计时。
- 运行时状态：`phase`、`elapsedMs`、`latchedRisks`、`lastControl`、`ego`。
- 派生事实：`timeout=true/false`、风险原因集合、钳制后的控制值。
- 展示状态：HUD 风险标签、结果面板失败原因、控制区连续可操作性。

## 6. 范围（In Scope）

- 接通 `session-controller` 到 HUD/结果面板的风险与结束展示。

- 逻辑层与 UI 展示层必须同步，确保风险“可见且可验证”。

## 10. 验收标准（行为式）
1. Given 会话在 `RUNNING`，When 发生碰撞，Then HUD 立即显示碰撞风险且该风险在本局内保持锁存。  
2. Given 会话在 `RUNNING`，When 风险已出现后车辆回到安全区域，Then HUD 仍保留已锁存风险直到本局结束或重试。  
3. Given 会话在 `RUNNING` 且达到超时阈值，When 系统轮询到超时，Then 会话自动进入 `SETTLING -> DONE`，结果面板显示超时失败。  
4. Given 输入 `throttle=1.4` 或 `-1.2`，When 输入被处理，Then 实际值钳制到 `1/-1` 且会话保持 `RUNNING` 不中断。  
5. Given 会话已 `DONE` 且用户点击重试，When 新一局开始，Then 上一局锁存风险被清空。

## 11. 测试策略（有预算）
- 单元测试：`risk-recorder` 锁存逻辑、`input-guard` 钳制逻辑、`session-end-policy` 超时判定。
- 状态机测试：超时自动触发 `RUNNING -> SETTLING -> DONE`，以及重试后风险清空。
- 集成 smoke：
- “运行中触发风险 -> HUD 锁存展示 -> 结束展示原因”。
- “超时自动结束 -> 结果面板显示超时”。
- 本卡不测：四场景循环完整性与视觉差异完备性。

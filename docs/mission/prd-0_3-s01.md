# 任务卡 Slice s01 - 基础场景语义渲染首屏可见

## 1. 本卡要交付的用户能力
用户在练习页选择 `normal-reverse-parking` 后，系统在主画布展示黑色道路、白色标线、停车位划线、绿化带、静态参照车。

## 2. 本卡为什么现在做
- 先补齐“看得见且可识别”的训练语义环境，避免后续控制和判定在空白场景上验证。
- 降低 first playable 阶段最大风险：链路有交互但场景不可见。
- 本卡对应 first playable 的“场景首屏可见”段。

## 3. 触达的闭环链路
- Trigger: 进入练习页并选择场景。
- Input: `scenarioId`。
- Runtime/State: `IDLE -> READY` 并挂载渲染结果。
- Core Logic: `play-page -> scene-loader -> scene-renderer`。
- Derived Facts / Settlement: 生成图层顺序化输出，暂不进入结算。
- UI Output: 主画布可见基础语义图层。

## 4. 影响的状态机
- 当前涉及状态：`IDLE`、`READY`。
- 新增/修改的状态迁移：不新增状态；增强 `IDLE -> READY` 时的场景加载与渲染副作用。
- 不在本卡处理的状态：`RUNNING`、`SETTLING`、`DONE` 的规则细化。

## 5. 影响的数据流
- 输入数据：场景选择输入。
- 运行时状态：`selectedScenario`、`renderReady`、`renderedScene`。
- 派生事实：有序图层列表与可见层集合。
- 展示状态：画布语义元素与图层顺序。

## 6. 范围（In Scope）
- 新增场景渲染契约与样式 token。
- 新增 `scene-loader` 负责场景渲染规格装载。
- 新增 `scene-renderer` 负责输出冻结图层顺序的渲染结果。
- 在 `play-page` / `session-controller` 接通场景选择到渲染结果。

## 7. 非范围（Out of Scope）
- 不实现车辆动态外观渲染。
- 不实现风险锁存、超时策略、最终分类判定。
- 不新增生产依赖，不改 auth/infra。

## 8. 实现边界 / 约束
- 不改动 `auth/` 和 `infra/`。
- 不新增生产依赖。
- 只做必要接线，不做无关重构。
- 图层顺序冻结为：`background -> road -> laneMarkings -> parkingSlots -> decorations -> staticCars -> egoCar -> HUDOverlay`。

## 9. 建议改动模块
- `src/features/parking-contract/contract-constants.ts`：渲染契约、图层顺序、样式 token。
- `src/scene-loader.ts`：场景渲染规格装载。
- `src/scene-renderer.ts`：按冻结顺序生成渲染输出。
- `src/play-page.ts`：编排场景选择链路。
- `src/session-controller.ts`：保存渲染状态到会话视图。

## 10. 验收标准（行为式）
1. Given 会话在 `IDLE`，When 用户选择 `normal-reverse-parking`，Then 相位变为 `READY` 且 `renderReady=true`。
2. Given 已完成场景渲染，When 检查可见图层，Then 顺序为 `background -> road -> laneMarkings -> parkingSlots -> decorations -> staticCars`。
3. Given 已选中有效场景，When 输入非法 `scenarioId`，Then 当前有效场景保持不变且会话不中断。

## 11. 测试策略（有预算）
- 单元测试：场景装载映射、图层顺序输出、可见层集合。
- 状态机测试：`IDLE -> READY` 的渲染副作用。
- 集成 smoke：选择场景后画布语义元素可见；重复选择不叠加重复图层。
- 本卡不测：结算判定、风险语义、超时逻辑。

## 12. 实施步骤
- Step 1：在契约层定义渲染相关类型与 token。
- Step 2：实现 `scene-loader` 和 `scene-renderer`。
- Step 3：接通 `play-page -> session-controller` 的渲染状态。
- Step 4：补充并执行测试与 smoke 清单验证。

## 13. 风险与回滚
- 主要风险：渲染职责与会话职责耦合，影响后续切片拆分。
- 回滚边界：仅回滚渲染契约与接线文件，不动输入/结算逻辑。
- 无破坏确认：既有输入归一化与结束占位逻辑测试保持通过。

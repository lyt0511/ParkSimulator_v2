# 停车模拟器需求草稿 v0.1（前半部分）

## 1. 目标（Goal）
把新手司机在停车练习中的“凭感觉判断是否能停进去”，改善为“可通过模拟操作得到可视化过程反馈与最终可入位结论”，用于快速验证停车思路。
原型目标：可跑通、基本功能正确。

## 2. 角色（Role）
新手司机（单人使用）。

## 3. 触发条件（Trigger）
用户在想验证某个停车思路时，可随时启动系统。
一次模拟在用户手动点击“完成/判定”时结束并输出最终结果。

## 4. 输入（Input）
场景输入（四选一）：
- 正常倒车入库
- 窄路倒车入库
- 正常侧方停车
- 窄路侧方停车

控制输入（混合）：
- 方向盘：离散输入（左/右/回正）
- 动力：连续输入（前进/后退强度）

## 5. 输出（Output）
仅前端界面展示，不生成文件。

过程输出（实时）：
- 车辆相对车位的位置与朝向
- 风险标记：是否压线、是否碰撞

结束输出（用户手动结束后）：
- 是否成功停进车位（是/否）

## 6. 判定与验收口径（当前已锁定）
成功判定：
- 车身包围盒完全在车位框内
- 最终速度为 0
- 压线/碰撞在过程中仅记录风险，不立即终止；最终统一判定

原型验收：
- 四个场景均可完整跑通
- 预设测试用例中，系统最终判定与人工预期一致率 >= 90%

## 7. 当前假设
- 先不扩展多角色（如教练/管理员）。
- 先不要求导出报告、历史记录、复杂评分体系。
- 以“练习验证”而非“驾考严格评分”为产品定位。

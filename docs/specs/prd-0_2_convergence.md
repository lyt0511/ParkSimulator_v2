# 停车模拟器需求收敛补充 v0.2

## 边界与异常
- 挑战：当前“包围盒入框+速度为0=成功”会允许“发生碰撞后仍显示成功”，与“帮助新手形成正确停车判断”存在冲突。
- 已明确边界：仅 4 个场景、单用户、手动结束、仅前端展示、不导出文件。
- 未明确项：碰撞/压线如何影响最终结果。
  - 默认建议值：输出双结果（入位是否成功 + 安全是否合规）。
  - 理由：保留练习连续性，同时避免安全误导。
- 未明确项：长时间未结束会话如何处理。
  - 默认建议值：300 秒超时自动结束并标记“未完成”。
  - 理由：避免会话悬挂，便于验收与回归。
- 未明确项：车辆出界/卡死如何处理。
  - 默认建议值：标记异常并允许一键重置场景。
  - 理由：原型需快速恢复，而非重启系统。
- 未明确项：控制输入异常（越界、非法组合）如何处理。
  - 默认建议值：动力钳制到 [-1,1]，方向仅允许左/右/回正。
  - 理由：简化异常分支并稳定判定。
- 未明确项：场景初始位姿是否随机。
  - 默认建议值：每场景固定 1 套初始位姿。
  - 理由：保证结果可复现。

## 约束
- 挑战：当前约束偏“功能跑通”，缺少“判定口径固定”的工程约束。
- 已明确约束：原型优先、基本正确、4 场景固定、无文件输出。
- 未明确项：运行平台范围。
  - 默认建议值：桌面 Chrome 最新稳定版。
  - 理由：先收敛兼容面，降低输入事件差异。
- 未明确项：车辆/车位/道路参数管理。
  - 默认建议值：参数写死为配置常量，不开放 UI 改参。
  - 理由：先锁定验收口径。
- 未明确项：仿真模型复杂度。
  - 默认建议值：低速运动学模型。
  - 理由：满足原型目标，投入最低。
- 未明确项：数据存储策略。
  - 默认建议值：仅内存态，不持久化。
  - 理由：符合“只在前端展示”的边界。

## 非功能
- 挑战：若不定义非功能，可能出现“功能正确但不可用/不可测”。
- 未明确项：实时反馈频率。
  - 默认建议值：状态刷新 >= 20Hz。
  - 理由：满足“过程判断”可感知。
- 未明确项：交互延迟。
  - 默认建议值：控制到状态变化 < 100ms。
  - 理由：避免用户误判动作效果。
- 未明确项：渲染流畅度。
  - 默认建议值：常见笔记本 FPS >= 30。
  - 理由：保证可操作性。
- 未明确项：稳定性。
  - 默认建议值：连续运行 30 分钟无崩溃。
  - 理由：满足演示与基础验证。
- 未明确项：结果一致性。
  - 默认建议值：同输入序列重复运行结果一致。
  - 理由：支撑一致率验收。
- 未明确项：验收样本规模。
  - 默认建议值：至少 40 条预设用例（每场景 10 条）。
  - 理由：90% 一致率需足够样本。

## 非目标
- 挑战：不声明非目标会触发范围蔓延，拖慢原型闭环。
- 非目标：不做 AI 驾驶建议/自动教练。
  - 默认建议值：v0.1 不包含该能力。
  - 理由：当前只承诺状态与结果判断。
- 非目标：不做驾考全规则严格评分。
  - 默认建议值：保持练习验证定位。
  - 理由：与既定目标一致。
- 非目标：不做账号、历史记录、报表导出。
  - 默认建议值：全部延后。
  - 理由：与“仅前端展示”一致。
- 非目标：不做天气/车型/复杂交通参与者。
  - 默认建议值：全部不纳入 v0.1。
  - 理由：先聚焦停车闭环。
- 非目标：不做移动端与外设适配。
  - 默认建议值：先键鼠桌面。
  - 理由：降低适配复杂度。

## 需立即决策项
- 决策项：最终输出是一维“是否停进”还是二维“入位+安全”。
  - 默认建议值：二维输出。
  - 理由：立即消除安全语义冲突。
- 决策项：碰撞/压线是否硬失败。
  - 默认建议值：不硬中断，但安全维度判失败。
  - 理由：兼顾训练连续性与安全反馈。
- 决策项：成功判定容差（边界/速度抖动）。
  - 默认建议值：边界容差 0.05m，速度阈值 0.05m/s。
  - 理由：避免数值噪声误判。
- 决策项：会话结束策略。
  - 默认建议值：手动结束 + 300 秒超时。
  - 理由：防止无穷会话。
- 决策项：场景参数是否固定。
  - 默认建议值：固定参数。
  - 理由：保证可复现验收。
- 决策项：一致率验收样本口径。
  - 默认建议值：40 条预设用例，人工双人复核标签。
  - 理由：保证 90% 指标可信。

## 可延后项
- 延后项：是否给“下一步动作建议”。
  - 默认建议值：v0.1 不做。
  - 理由：不影响当前闭环成立。
- 延后项：细粒度评分（平顺性、修正次数、耗时）。
  - 默认建议值：v0.2 再定。
  - 理由：可在闭环后增量加入。
- 延后项：多角色能力（教练/管理员）。
  - 默认建议值：保持单角色。
  - 理由：当前用户画像单一。
- 延后项：历史回放与持久化。
  - 默认建议值：暂不做。
  - 理由：与“前端即时展示”一致。
- 延后项：输入设备扩展（手柄/方向盘外设）。
  - 默认建议值：先键鼠。
  - 理由：先降低适配复杂度。
- 延后项：高拟真物理模型。
  - 默认建议值：保持低速运动学。
  - 理由：原型阶段收益低于成本。
